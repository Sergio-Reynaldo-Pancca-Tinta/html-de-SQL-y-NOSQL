Introducción a las Bases de Datos SQL

Las bases de datos SQL, también conocidas como bases de datos relacionales, son sistemas que organizan la información de manera estructurada y tabular, priorizando la integridad de los datos y la consistencia.

Estos sistemas almacenan datos en tablas compuestas por filas y columnas, donde cada fila representa un registro y cada columna representa un atributo de ese registro.

Modelo y Estructura Relacional

El modelo relacional fue propuesto por Edgar Frank Codd en 1970 como una solución a los problemas inherentes de las bases de datos de red y jerárquicas.

Componentes principales:

Tablas

Estructuras que contienen los datos organizados en filas y columnas, similares a hojas de cálculo.

Filas (Registros)

Cada fila representa una entidad o elemento único en la tabla.

Columnas (Atributos)

Cada columna representa una característica o propiedad de las entidades.

Ejemplo de una tabla:

ID Nombre Email Edad
1 Juan Pérez juan@email.com 28
2 María García maria@email.com 32
3 Carlos López carlos@email.com 25

Esquema Fijo y Predefinido

Las bases de datos SQL tienen un esquema fijo y predefinido, lo que significa que la estructura de la tabla (columnas y sus tipos de datos) debe definirse antes de insertar los datos.

Nota sobre la flexibilidad del esquema

Aunque algunas fuentes mencionan que SQL posee un "esquema flexible", esta afirmación contradice el consenso general. La flexibilidad de esquema es, de hecho, una característica distintiva de las bases de datos NoSQL.

Ventajas del esquema predefinido:

  • Mayor integridad de datos
  • Mejor rendimiento para aplicaciones transaccionales
  • Validación de tipos de datos
  • Estructura consistente y predecible

Integridad de Datos y Normalización

Una de las principales prioridades de las bases de datos SQL es la integridad de los datos. Para lograr esto:

  • La información se almacena en tablas separadas para cada entidad
  • Las relaciones entre tablas se establecen mediante claves foráneas (Foreign Keys)
  • Se aplica el proceso de normalización para evitar anomalías, reducir la redundancia y proteger la integridad

Normalización

La normalización implica organizar los datos para cumplir con las formas normales. Se recomienda aplicar al menos hasta la tercera forma normal:

Primera Forma Normal (1FN)

Todos los campos deben ser atómicos (almacenar un solo valor) y la tabla debe tener una clave primaria.

Segunda Forma Normal (2FN)

Debe estar en 1FN y todos los campos no clave deben depender completamente de la clave primaria.

Tercera Forma Normal (3FN)

Debe estar en 2FN y eliminar las dependencias transitivas (los campos no clave no deben depender de otros campos no clave).

Lenguaje de Consulta Estructurado (SQL)

SQL es un lenguaje declarativo de "alto nivel" que se utiliza para consultar, manipular y controlar los datos en bases de datos relacionales.

Componentes del SQL:

DDL (Data Definition Language)

Para definir la estructura de la base de datos: crear, modificar o eliminar tablas y otros objetos.

CREATE TABLE Clientes (
  id INT PRIMARY KEY,
  nombre VARCHAR(50),
  email VARCHAR(100)
);

DML (Data Manipulation Language)

Para manipular la información: seleccionar, insertar, actualizar y eliminar datos.

SELECT nombre, email FROM Clientes
WHERE edad > 25;

DCL (Data Control Language)

Para gestionar roles y permisos de acceso a la base de datos.

GRANT SELECT, INSERT
ON Clientes
TO usuario1;

Propiedades ACID

Las bases de datos SQL se adhieren estrictamente a las propiedades ACID, que garantizan que las transacciones se procesen de manera precisa y confiable.

Atomicidad (Atomicity)

Las transacciones son "todo o nada": se completan todas las operaciones o ninguna.

Consistencia (Consistency)

Cada transacción lleva la base de datos de un estado válido a otro estado válido.

Aislamiento (Isolation)

Las transacciones concurrentes no interfieren entre sí.

Durabilidad (Durability)

Los cambios confirmados persisten incluso ante fallos del sistema.

Escalabilidad y Rendimiento

Escalabilidad

Las bases de datos SQL generalmente emplean escalabilidad vertical, que implica aumentar los recursos de un único servidor (CPU, RAM, etc.).

El escalamiento horizontal (fragmentación) es posible, pero puede ser más costoso o complejo en comparación con NoSQL.

Rendimiento

Están optimizadas para consultas y transacciones complejas. El motor de la base de datos utiliza un optimizador de consultas para determinar la forma más eficiente de ejecutar una sentencia SQL.

Casos de Uso Típicos

Las bases de datos SQL son ideales para sistemas donde la integridad y consistencia de los datos son críticas:

  • Sistemas financieros (bancos, procesamiento de pagos)
  • Sistemas ERP (Planificación de Recursos Empresariales)
  • Sistemas CRM (Gestión de Relaciones con Clientes)
  • Comercio electrónico
  • Aplicaciones de reservas (hoteles, vuelos)
  • Escenarios que requieren un uso extensivo de uniones y agregaciones para informes y análisis
